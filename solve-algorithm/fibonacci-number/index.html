    ⚡ 문제 링크(프로그래머스)



    문제


    문제 풀이

    int 자료형의 범위는 c++에서 32비트 기준으로 -2,147,483,648 ~ 2,147,483,647 이며 실제로 2,147,483,647 에 +1을 해주게 되면 2,147,483,648이 아닌 -2,147,483,648 이 된다.

    이러한 자료형 범위 문제 때문에 문제에서는 친절히 1234567로 나눈 나머지 값을 이용 하라고 나와 있다.

    간단한 피보나치 수를 구현 해보는 것이었기 때문에 문제에 주어진 대로 재귀함수를 이용하여 문제를 풀이 했다.


    📃 틀린 코드

    #include <string>
#include <vector>

using namespace std;

int Fibonacci(int num){
    if(num == 0) return 0;
    else if(num == 1 )return 1;
    else return (Fibonacci(num - 1) + Fibonacci(num - 2)) % 1234567;
}

int solution(int n) {
    int answer = 0;
    answer = Fibonacci(n);
    return answer;
}

    ❗ 채점결과 :

    !!아니!! 이게 뭔가!!

    재귀함수로 문제를 풀이했기 때문에 숫자(n)이 커지면서 재귀에 재귀가 실행되는 횟수가 많아지며 재귀지옥이 되어 아래의 사진과 같이 테스트 7이 넘어가면서부터 시간 초과로 실패하였다.


    📃 수정 코드 ( 최종 답 )

    #include <string>
#include <vector>

using namespace std;

int solution(int n) {
    int answer = 0;
    int F[100001];
    F[0] = 0; F[1] = 1;
    
    for(int i=2; i<=n; i++){
        F[i] = (F[i-1] + F[i-2] )  % 1234567;
    }
    answer = F[n];
    return answer;
}

    재귀함수 때 처럼 F[4]를 구하기 위하여 F[3], F[2]를 재귀함수를 통해 다시 또 구하는 식이 아니라 배열을 통해 앞에서 구한 값인 F[3], F[2]는 참조를 통해 바로 계산하는 방법으로 구현 하였다.
