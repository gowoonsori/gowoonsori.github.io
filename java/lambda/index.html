    람다식은 익명함수를 지칭하는 용어로 함수 지향적인 표현에 가까운 표현식이다.

    람다식은 런타임시에 익명 구현 객체를 생성하여 사용한다.

    코드의 간결성, 지연 연산을 통한 성능 향상, 이터레이션 코드를 구현하는데 있어 불필요한 부분들을 제거할 수 있는 장점이 있다.

    하지만, 모든 원소를 순회하는 경우 일반 반복문보다 조금 더 느리고, 디버깅 시에 함수 콜스택 추적이 어렵고, 남용할 경우 오히려 코드 읽기 어려워 질 수 있다.



    람다식 사용법

    기본적인 구문은 (type arg1, type arg2...) -> { 실행 구문 }; 이고, 보다 간결하게 표현하기 위해 다음과 같은 특징을 갖고 표현할 수 있다.

    • arg1…와 같은 매개변수의 type은 생략이 가능하다.

    • 매개변수가 하나라면 괄호 ()를 생략이 가능하다.

    • 실행 구문이 하나라면 중괄호 {}와 return을 생략이 가능하다.


    for문을 이용하여 0~9까지 출력하는 구문을 람다식을 이용하면 아래와 같이 한줄로 줄여 표현할 수 있다.

    public class LambdaTest {
    @Test
    void basicLoop(){
        for (int i = 0; i < 10; i++) {
            System.out.println(i);
        }
    }
}

    @Test
    void LambdaLoop(){
        IntStream.range(0, 10).forEach((int value) -> System.out.println(value));

        IntStream.range(0, 10).forEach((value) -> System.out.println(value));  //컴파일러의 추론을 통해 자료형 생략

        IntStream.range(0, 10).forEach(value -> System.out.println(value));  //매개변수가 하나일때 괄호 생략

        IntStream.range(0, 10).forEach(System.out::println); //매서드 레퍼런스 사용
    }



    함수형 인터페이스

    public class LambdaTest {
  @Test
  void RunnableTest(){
    Thread th1 = new Thread(new ExampleThread());  //일반적인 방법

    Thread th2 = new Thread(()->System.out.println("thread2")); //람다식 표현 방법

    th1.start();
    th2.start();
  }

  class ExampleThread implements Runnable{
    @Override
    public void run(){
      System.out.println("Thread 1");
    }
  }
}

    추상 메서드가 하나만 정의되어있는 인터페이스를 함수형 인터페이스(Functional Interface라고 하고, 10주차 스터디 주제였던 Thread 에서 나온 Runnable이 그 예시이다.

    ◾ 함수형 인터페이스 선언

    @FunctionalInterface
public interface ExampleFunctionalInterface {
    public void sayHello(String str);
}

    일반적인 인터페이스 선언 방식에 추상 메서드가 한개만 존재한다면 그것이 함수형 인터페이스이고, 컴파일 타임시 에러를 출력할 수 있게 @FunctionalInterface어노테이션을 붙여주면 추상메서드 두개 선언시 에러를 출력한다.



    Variable Capture

    람다식은 람다식 외부의 final 지역 변수 또는 final이 아니지만 딱 한번만 초기화 된 변수에 접근할 수 있다.

    ◾ final 지역 변수에 접근

    public class Main {
  public static void main(String[] argv) {
    final String x = "Hello";
    Function<String,String> func1 = str -> {return str + " "+ x ;};
    System.out.println(func1.apply("Lambda Study"));
  }
}

    ◾ final이 아니고 한번만 초기화 된 변수에 접근

    public class Main {
  public static void main(String[] argv) {
    String x = "Hello";
    Function<String,String> func1 = str -> {return str + " "+ x ;};
    System.out.println(func1.apply("Lambda Study"));
  }
}

    ◾ 람다식 외부에서 정의된 값을 변경할 수는 없다.

    public class Main {
  public static void main (String [] argv) {
    String x = "Hello" ;
    Function <String,String> func1 = str-> { x = "a"; return str + "" + x;};  //compile error
    System.out.println (func1.apply ( "Lambda Study" ));
  }
}

    ◾ 람다식 외부의 지역변수가 아니라면 변경 가능하다.

    public class Main {
  String x = "Hello" ;
  public static void main (String [] argv) {
    Function <String,String> func1 = str-> { x = "Hi"; return str + "" + x;};
    System.out.println (func1.apply ( "Lambda Study" ));
  }
}



    메소드, 생성자 레퍼런스

    메소드, 생성자 레퍼런스는 람다식을 더 간단하게 표현하는 방법이로 함수의 축약형이라고 할 수 있다.
    명시적으로 메서드를 참조함으로써 가독성을 높이고, 람다를 편하게 사용할 수 있게 해준다.

    람다식 사용법 설명에서 사용했던 System.out::println과 같은 방법으로 ClassName::MethodName의 형식으로 사용하게 된다.

    Consumer<String> func = str -> System.out.println(str);
=>
Consumer<String> func = System.out::println;

    위의 코드와 같이 축약해 사용할 수 있으며, String 매개변수 한개를 받아 void를 출력시키는 함수라는 의미가 생략되어있다.
    때문에 사용하려는 메서드의 매개변수와 리턴 타입을 잘 알고 있어야 사용이 가능하다.

    ◾ Static 메소드

    public class LambdaTest {
  public static void sayHello(String name){
    System.out.println("Hello" + name);
  }

  @Test
  void methodReferenceTest(){
    Consumer<String> consumer = LambdaTest::sayHello;
    //Consumer<String> consumer = str -> LambdaTest.sayHello(str);
    consumer.accept("홍길동");
  }

    여기서 Consumer<T> 자바에서 제공하는 함수형 인터페이스로 T 타입 매개변수를 받아 void를 리턴하는 함수형 인터페이스이다.

    주석처리된 람다식을 메서드 레퍼런스를 이용해 더 축약해 사용할 수 있게 되었다.

    ◾ Instance 메서드

    public class User {
    String name;
    public User(String name){this.name = name;}
    public void sayHello(){
        System.out.println("Hello " + name);
    }
}

public class LambdaTest {
  @Test
  void methodReferenceTest(){
    Consumer<User> consumer = User::sayHello;
    consumer.accept(new User("홍길동"));
  }
}

    첫번째와 같은 static 메서드가 아닌 인스턴스 객체의 메서드를 참조하는 방식으로 Consumer의 매개변수로 User클래스를 주어 다음과 같이 사용할 수 있다.

    public class User {
    String name;
    public User(String name){this.name = name;}
    public void sayHello(){
        System.out.println("Hello " + name);
    }
}

public class LambdaTest {
  @Test
  void methodReferenceTest(){
    List<User> users = Arrays.asList(new User("고길동"), new User("둘리"), new User("또치"));
    users.stream().forEach(User::sayHello);
    //users.stream().forEach(user -> user.sayHello());
  }
}

    이를 조금더 응용해보면 배열의 forEach와 함께 사용한다면 더욱더 축약해서 표현이 가능하다.

    ◾ 생성자 (Constructor) 메서드

    생성자 메서드를 참조하여 축약해서 표현할 수 있게 한 방법이다.

    public class User {
    String name;

    public User(String name){this.name = name;}
}

public class LambdaTest {
  @Test
  void methodReferenceTest(){
    User user = new User("홍길동");

    Function<String, User> userConstructor = name -> new User(name);

    Function<String, User> userConstructor1 = User::new;
    User user2 = userConstructor1.apply("홍길동");
  }
}

    Function 함수형 인터페이스를 이용하여 User의 생성자 메서드를 특정 변수에 할당해줄 수 있고 new키워드가 없어도 apply()메서드를 통해 생성자 메서드처럼 사용이 가능하다.

    Fucntion<T,R>은 Consumer와 마찬가지로 자바에서 제공해주는 함수형 인터페이스로 T타입 매개변수를 받아 R타입을 return하는 함수형 인터페이스이다.



    자주 사용하는 함수형 인터페이스

    자바에서 제공하는 함수형 인터페이스가 존재하는데 바로 위에서 사용했던 Consumer, Function과 멀티 프로그래밍의 Runnable등 이 존재한다.

    함수형 인터페이스 반환 타입 추상 메서드 이름 다른 메서드 기능
    Runnable void run 없음 코드 수행
    Supplier T타입 get 없음 T타입 값 return
    Consumer void accept andThen T타입 값을 파라미터로 이용
    BiConsumer<T,U> void accept andThen T,U 타입 값을 파라미터로 이용
    Function<T,R> R apply compose, andThen, identity T타입 값을 파라미터로 이용하고 R타입 return
    BiFunction<T,U,R> R apply andThen T,U타입 값을 파라미터로 이용하고 R타입 return
    UnaryOperator T apply compose, andThen, identity T타입 값을 파라미터로 이용하고 T타입 return
    BinaryOperator<T,T> T apply andThen, maxBy, minBy T타입 값 두개의 파라미터를 이용하고 T타입 return
    Predicate boolean test and, or, negate, isEqual T타입 값을 파라미터로 이용하고 boolean return
    BiPredicate<T,U> boolean test and, or, negate T,U타입 값을 파라미터로 이용하고 boolean return
