    노드 (Node.js) 설치하기

    사용할 IDE는 vscode를 사용할 예정이다.

    react 프로젝트를 시작하기에 앞서 js를 이용하여 웹프로그래밍을 하기 위해 노드를 설치 해주어야 한다.

    다운로드 링크 : https://nodejs.org/ko/

    위의 링크를 통해 다운 및 설치를 해주었다.

    터미널에 npm -v를 통해 설치가 되었는지/무슨 버전을 설치했는 지 확인할 수 있다.



    프로젝트 폴더 만들기

    프로젝트를 진행할 earn-points-web이라는 폴더를 만들고 이 폴더 안에서 front / back을 나누어 개발을 할 것이고 front먼저 틀을 잡아놓을 계획이기 때문에 front먼저 생성 해주었다.

    또한 react의 SSR을 위해 Next.js를 사용해 줄 생각이기 때문에 next.js 플러그인을 아래의 방법대로 설치를 진행해 주었다.

    • mkdir front를 통해 폴더 생성
    • cd front
    • npm init -y : npm 초기화
    • npm install --save react react-dom next : next 설치

    설치를 진행하고 나면 package.json이 생기는 데 여기에 각종 설정을 추가해주었다.

        "name": "front",
    "version": "1.0.0",
    "private": true,
    "description": "",
    "main": "index.js",
    "author": "Euisung",
    "license": "ISC",
    "scripts": {
        "dev": "next -p 3060",
        "build": "next build",
        "start": "next start",

    author 과 scripts와 같은 정보를 추가해주었다.

    main옵션을 보면 index.js가 root page가 되는 것을 볼 수 있다.

    여기서 dev는 next를 localhost로 실행시켜보는 script명령어이며 -p옵션을 통해 port 번호를 설정해 줄 수도 있다.



    index.js 작성

    front폴더 안에 pages폴더를 새로 만들고 index.js를 만들어 아래의 코드를 작성해 주고 정상적으로 Next.js가 잘돌아가는지 확인했다.

    const Home = () => {
  return <div>Hello Next.js</div>;
};

export default Home;



    localhost:3060 으로 접속하니 사진과 같이 실행이 정상적으로 되었다.



    _app.js 작성

    페이지들이 바뀌어도 기본적인 layout의 유지를 위해 _app.js를 아래와 같이 작성해 주었다.

    import React from 'react';
import Head from 'next/head';

const EarnPointsWeb = ({Component}) => {
  return (
    <>
      <Head>
        <meta charSet="utf-8" />
        <title>포인트 적립</title>
      </Head>
      <Component />
    </>
  );
};

export default EarnPointsWeb;



    아래의 사진과 같이 index.js를 _app.js 가 감싸주어 타이틀이 변한 것을 볼 수 있다.


    _app.js는 다음과 같은 기능을 수행 하기 위해 사용된다.

    • 공통 layout부분 유지
    • 페이지를 네비게이팅(navigating)할 때 상태 유지
    • graphQL과 같은 추가 데이터들을 페이지들에 사용
    • componentDidCatch로 에러 관리



    Redux 와 Redux-saga 설치

    Redux란 한마디로 말해 중앙 데이터 저장소역할을 해주는 라이브러리이다.
    각 컴포넌트, 페이지들의 데이터들을 서로 교환하는 방식이 아닌 중앙 데이터에서 모든 데이터를 관리하고 보내주는 것이 훨씬 효율적이기 때문에 사용할 생각이다.

    Redux를 사용하게 되면 변경사항마다 action을 작성해주어야 하기 때문에 코드가 길어질 수 있지만 중요한 개념으로써 한번 사용해 볼 생각이다.
    Redux말고도 Mobax, Context API 같은 라이브러리도 있다.

    Redux는 모든 history가 기록되기 때문에 에러 추적이 쉽다. Mobax는 코드량이 적어 생산성을 높일 수 있지만 상태 추적이 어렵고 Context API는 앱 규모가 작을때 권장 된다.


    설치 순서와 파일 수정

    npm i redux next-redux-wrapper을 통하여 설치를 진행했다.
    여기서 redux-wrapper는 redux의 손쉬운 사용을 위해 설치했다.


    • front/store/configureStore.js 파일 생성
    import {createWrapper} from 'next-redux-wrapper';
import {createStore} from 'redux';

import reducer from '../reducers';

const configureStore = () => {
  const store = createStore(reducer);
  return store;
};

const wrapper = createWrapper(configureSotre, {
  debug: process.env.NODE_ENV === 'development,',
});

export default wrapper;

    파일 생성 후 reducer에게 상태를 넘겨주고 페이지들을 redux로 감싸주기 위해 위의 코드를 작성해주었다. ( 아직 별 페이지도 없기 때문에 뭔가를 넘겨주지는 않는다…)

    redux는 action이 날라오면 reducer가 store의 state를 변경시키는 방식으로 동작


    • front/reducers/index.js 파일 생성
    import {HYDRATE} from 'next-redux-wrapper';
import {combineReducers} from 'redux';

const rootReducer = (state, action) => {

    case HYDRATE:
      return action.payload;

    • _app.js 수정
    import React from 'react';
import Head from 'next/head';
import wrapper from '../store/configureStore.js';

const EarnPointsWeb = ({Component}) => {
  return (
    <>
      <Head>

export default wrapper.withRedux(EarnPointsWeb);

    _app.js의 함수를 redux를 통하여 감싸주어 프로젝트의 모든 컴포넌트와 페이지에 적용시키기



    ant design 라이브러리 설치

    목적은 토이프로젝트의 빠른 개발을 통해 개념 공부이기 때문에, 디자인에 힘을 덜 들이고자 컴포넌트 UI 라이브러리 중 하나인 antd을 이욯하여 개발하기로 했다.

    아래의 공식문서에 각종 컴포넌트들 사용 법들이 잘 나와있어 가져다가 입맛대로 바꿔 사용하면 된다.

    그 외 플러그인들 설치

    redux와 더불어 api 요청을 saga를 이용하여 구현해보고자 redux-saga를 설치해 주었다.

    redux-saga란 redux-thunk다음으로 가장 많이 사용되는 라이브러리로 redux-thunk는 함수를 디스패치 할 수 있게 해주는 미들웨어이고, redux-saga는 모니터링하다 특정 액션이 발생하게 되면 특정 작업을 수행할 수 있게 해준다.
    (특정 수행은 js코드를 수행하는 것일 수도 있고 dispatch, 상태를 불러오는 것이거나, 비동기 작업시 기존 요청을 취소처리, api 요청 실패시 재요청 작업 등을 수행 할 수 있다.)

    chrome의 React Developer Tools 확장 프로그램과 더불어 개발시 redux 상태를 쉽게 파악하여 개발을 도와주는 redux-devtools-extension도 설치 해주었다.

    code convention을 지키기 위해 eslint , prettier를 설치 해주었다.

    code convention : 읽고 관리하기 쉬운 코드를 작성하기 위한 코딩 스타일 규약 -> 코드의 가독성이 좋아지고 오류의 발생요소를 줄여준다.
