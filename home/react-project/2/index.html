    가장 먼저 vscode를 통해 wsl2에 접속하고 wsl2에 폴더를 만들어 주었다.

    mkdir front
cd front
npm init -y
npm install --save react react-dom next

    만든 프로젝트안에 front라는 폴더를 만들어 react와 next의 의존성을 추가해주었다.

    "scripts": {
        "dev": "next -p 3060",
        "build": "next build",
        "start": "next start",
    .
    .
    .

    package.json에 프로젝트 실행을 보다 간편하게 하기 위해 명령어를 수정해주었다.

    npm run dev를 통해 개발모드로 실행 시키고 npm run build와 npm start로 실제 동작을 시킬 수 있다. (test와 start 명령은 run을 생략할 수 있다.)



    index.js 작성

    프로젝트에 기본시작 파일은 pages/index.js가 되기 때문에 index.js를 다음과 같이 작성해 주었다.

    const Index = () => {
  return <div>Hello Next.js</div>;
};

export default Index;

    npm run dev를 통해 실행 시키면 잘 동작이 되는 것을 볼 수 있다.

    이때 나처럼 wsl2로 실행시킨 사람이라면 window의 브라우저에 localhost:3060을 타이핑해서 들어간다면 접속이 되지 않을 것이다.

    wsl2는 별도의 ip를 가지고 있기 때문인데, wsl2의 ip:3060으로 접속하거나 실행시킨 console창의 링크를 클릭하면 window의 브라우저에 포트번호가 변경되며 열릴 것이다.



    _app.js

    나는 왼쪽과 위쪽에 메뉴바가 고정으로 있는 디자인을 생각하고 있어 전체 layout은 유지하고 state를 유지하기 위해 모든 페이지에서 사용할 _app.js를 만들어 주었다.

    import React from 'react';
import Head from 'next/head';
import PropTypes from 'prop-types';

import { RecoilRoot } from 'recoil';

const PointsWeb = ({ Component }) => {
  return (
    <RecoilRoot>
      <Head>
        <title>포인트 적립</title>
      </Head>
      <Component />
    </RecoilRoot>
  );
};

PointsWeb.propTypes = {
  Component: PropTypes.elementType.isRequired,
};

export default PointsWeb;

    모든 page(일종의 Component)들이 Component라는 prop으로 이용하여 해당 컴포넌트를 다른 Layout(Component)로 감싸는 구조이다.

    또한 이 _app.js가 최상단 Component로 state을 모든 Component에서 사용하기 위해 로 이를 감싸 주었다.



    AppLayout

    모든 페이지에서 공통적으로 사용할 AppLayout을 정의한 컴포넌트이다.

    import React, { useState } from 'react';
import PropTypes from 'prop-types';
import Router from 'next/router';
import {
  AppBar,
  Drawer,
  CssBaseline,
  Divider,
  Hidden,
  IconButton,
  Toolbar,
  List,
  ListItemText,
  ListItem,
  makeStyles,
} from '@material-ui/core';
import MenuIcon from '@material-ui/icons/Menu';

const drawerWidth = 180;

const useStyles = makeStyles((theme) => ({
  root: {
    display: 'flex',
  },
  drawer: {
    [theme.breakpoints.up('sm')]: {
      width: drawerWidth,
      flexShrink: 0,
    },
  },
  appBar: {
    [theme.breakpoints.up('sm')]: {
      width: `calc(100% - ${drawerWidth}px)`,
      marginLeft: drawerWidth,
    },
  },
  menuButton: {
    marginRight: theme.spacing(2),
    [theme.breakpoints.up('sm')]: {
      display: 'none',
    },
  },
  menuText: {
    textAlign: 'right',
    paddingRight: theme.spacing(4),
  },
  toolbar: {
    paddingTop: '10px',
    ...theme.mixins.toolbar,
  },
  drawerPaper: {
    width: drawerWidth,
    background: '#284D7A',
    color: '#ffffff',
  },
  contentContainer: {
    flexGrow: 1,
    padding: theme.spacing(3),
  },
  content: {
    textAlign: 'center',
    width: '100%',
    height: '100%',
  },
}));

const AppLayout = ({ children }) => {
  const classes = useStyles();
  const [mobileOpen, setMobileOpen] = useState(false);
  const handleDrawerToggle = () => {
    setMobileOpen(!mobileOpen);
  };

  const drawer = (
    <div>
      <div className={classes.toolbar}>
        <ListItem button key="Home" className={classes.menuText}>
          <ListItemText
            primary="Home"
            onClick={() => {
              Router.push('/');
            }}
          />
        </ListItem>
      </div>
      <Divider />
      <List>
        {[
          { name: '로그인', link: 'login' },
          { name: '회원가입', link: 'signup' },
        ].map((text) => (
          <ListItem button key={text.name} className={classes.menuText}>
            <ListItemText
              primary={text.name}
              onClick={() => {
                Router.push(`${text.link}`);
              }}
            />
          </ListItem>
        ))}
      </List>
      <Divider />
      <List>
        {[
          { name: '포인트 적립', link: 'points' },
          { name: '고객 정보 보기', link: 'customers' },
        ].map((text) => (
          <ListItem button key={text.name} className={classes.menuText}>
            <ListItemText
              primary={text.name}
              onClick={() => {
                Router.push(`${text.link}`);
              }}
            />
          </ListItem>
        ))}
      </List>
    </div>
  );

  return (
    <div className={classes.root}>
      <CssBaseline />
      <AppBar position="fixed" className={classes.appBar} color="primary">
        <Toolbar>
          <IconButton
            color="inherit"
            aria-label="open drawer"
            edge="start"
            onClick={handleDrawerToggle}
            className={classes.menuButton}
          >
            <MenuIcon />
          </IconButton>
        </Toolbar>
      </AppBar>

      <nav className={classes.drawer} aria-label="mailbox folders">
        <Hidden smUp implementation="css">
          <Drawer
            variant="temporary"
            anchor="left"
            open={mobileOpen}
            onClose={handleDrawerToggle}
            classes={{ paper: classes.drawerPaper }}
            ModalProps={{ keepMounted: true }}
            color="secondary"
          >
            {drawer}
          </Drawer>
        </Hidden>
        <Hidden xsDown implementation="css">
          <Drawer classes={{ paper: classes.drawerPaper }} variant="permanent" color="secondary" open>
            {drawer}
          </Drawer>
        </Hidden>
      </nav>
      <main className={classes.contentContainer}>
        <div className={classes.toolbar} />
        <div className={classes.content}>{children}</div>
      </main>
    </div>
  );
};

AppLayout.propTypes = {
  children: PropTypes.node.isRequired,
};

export default AppLayout;

    Material Ui의 temporary Drawer를 이용했으며 여기서 Materal Ui의 makeStyles를 통한 css class 생성 구문을 처음봤고 공식 문서를 통해 기본 방법을 공부하느라 제법 삽질을 했다.

    API를 보면 제법 많은 메서드를 지원해주는데 그 중 makeStyles함수를 통해 class와 css를 객체로 만들고 이를 useStyles()를 통해 객체로서 접근이 가능한 방식이었다.

    그리고 이 Layout을 기본으로 안의 내용만 바꿀것이기 때문에 _app.js의 Children을 이 Layout의 적절한 위치에 배치해주기 위해 이 컴포넌츠에서도 _app.js의 Children을 prop으로 넘겨받아 배치 시켜주었다.

    Drawer의 각 Nav마다 해당하는 페이지로 바꿔주기 위해 위해 Router를 이용하여 라우팅하였다.

    _document.js

    Next를 이용하고 있어 SSR이기 때문에 새로고침이나 뒤로가기를 누르면 Material Ui가 렌더링이 되지 않는 문제점이 존재했다.

    그래서 SSR시에도 Material Ui를 렌더링을 시켜주어야했는데 _document.js를 이용했다.

    _document.js는 React의 index.html과 같은 파일로 SPA에서의 시작점이 되는 파일과 같다.

    지금과 같이 Custom할때 작성을 하며 작성을 안하면 Next.js가 기폰 값을 사용하여 실행시키게 된다.

    {
  "presets" : ["next/babel"],
  "plugins": [
    ["babel-plugin-styled-components",{
      "ssr" : true,
      "displayName" : true
    }]
  ]
}

    _document.js를 작성하기 전에 babel-plugin-components를 설치해주고 .babelrc파일을 다음과 같이 작성해서 SSR시 styled components를 적용하게 설정을 해주었다.

    import React from 'react';
import NextDocument, { Html, Head, Main, NextScript } from 'next/document';
import { ServerStyleSheet as StyledComponentSheets } from 'styled-components';
import { ServerStyleSheets as MaterialUiServerStyleSheets } from '@material-ui/core/styles';

export default class Document extends NextDocument {
  static async getInitialProps(ctx) {
    const styledComponentSheet = new StyledComponentSheets();
    const materialUiSheets = new MaterialUiServerStyleSheets();
    const originalRenderPage = ctx.renderPage;

    try {
      ctx.renderPage = () =>
        originalRenderPage({
          enhanceApp: (App) => (props) =>
            styledComponentSheet.collectStyles(materialUiSheets.collect(<App {...props} />)),
        });

      const initialProps = await NextDocument.getInitialProps(ctx);

      return {
        ...initialProps,
        styles: [
          <React.Fragment key="styles">
            {initialProps.styles}
            {materialUiSheets.getStyleElement()}
            {styledComponentSheet.getStyleElement()}
          </React.Fragment>,
        ],
      };
    } catch (error) {
      console.log(error);
    } finally {
      styledComponentSheet.seal();
    }
  }

  render() {
    return (
      <Html>
        <Head />
        <body>
          <Main />
          <NextScript />
        </body>
      </Html>
    );
  }
}

    styled components Sheets를 이용하여 material ui styleSheets를 감싸 렌더링시에 Material Ui도 렌더링되게 설정을 해주었다.



    Theme color 수정

    Materila Ui에서 color를 테마로 제공하는 컴포넌트들이 있는데 이는 일반 css 속성으로 변환이 되지 않아 변경하기 위해 공식 문서를 보고 조금 공부를 하였다.

    Material Ui에서 제공하는 AppBar나 Button의 색을 바꿔주기 위해서는 기본 제공하는 palette의 색을 바꿔주거나 생성해주고 ThemeProvider를 적용하고자하는 컴포넌트를 감싸 테마를 사용할 수 있었다.

    나는 전체적인 컬러감을 생각해두고 있었고 `컬러는 4,5개 내에서 웹을 다지인 할 것이기 때문에 global theme를 만들어 전체 컴포넌트에 적용하고자 했다.

    global Theme provider 파일 작성

    import React from 'react';
import PropTypes from 'prop-types';

import { createMuiTheme } from '@material-ui/core/styles';
import { ThemeProvider } from '@material-ui/styles';

const GlobalTheme = createMuiTheme({
  palette: {
    primary: {
      main: '#284D7A',
    },
    secondary: {
      main: '#20B8AA',
    },
    tertiary: {
      main: '#56F0B9',
    },
    quad: {
      main: '#E9EBED',
    },
  },
});

const GlobalThemeProvider = ({ children }) => <ThemeProvider theme={GlobalTheme}>{children}</ThemeProvider>;

GlobalThemeProvider.propTypes = {
  children: PropTypes.elementType.isRequired,
};

export default GlobalThemeProvider;

    4가지의 색을 미리 선언해놓고 이 컴포넌트를 루트에 씌워 모든 컴포넌트에서 사용할 수 있게 할 생각으로 똑같이 children을 prop으로 받아 이를 감싸도록 해주었다.

    _app.js 수정

    import React from 'react';
import Head from 'next/head';

    <AppBar position="fixed"  color="primary">
<Button variant="contained" color="secondary" >

    그러면 위와 같이 color 속성으로 테마를 적용할 수 있고 잘 적용 된 것을 볼 수 있다.
