    🔴 레드 블랙 트리 (Red Black Tree)

    BST (이진 탐색 트리)를 기반으로 둔 Tree.

    Tree의 Rebalancing 방법 중 하나로 balanced한 트리이다.

    각 노드는 값(key)말고도 색을 갖고 있으며, 색은 레드 or 블랙 2종류다.


    📌 Red Black Tree가 갖는 특성

    Root Property : 루트(root)노드는 블랙(black)이다.

    External Property : 모든 외부 노드 (external node)는 블랙이다.

    Depth Property : 모든 단말 노드(leaf node)의 경우 루트부터 외부 노드 까지 방문하는 블랙 노드의 수가 같다.

    Internal Property : 빨강 노드의 자식은 블랙이다.
    == No Double Red : 레드 노드는 두개가 연속해서 올 수 없다.


    📌 특징

    • BST의 모든 특징을 갖는다.

    • 노드의 자식이 없는 경우 자식을 가리키는 포인터는 NULL값을 저장 ( NULL을 leaf node로 간주)

    • 루트 노드 부터 단말 노드(leaf node)까지 모든 경로 중 최소 경로와 최대 경로의 크기 비율은 2보다 크지 않다. ( balanced 상태 )

    • AVL Tree 에 비해 덜 깐깐하기에 탐색속도는 느릴 수 있으나, 삽입 / 삭제 속도는 더 빠르다.

    • 삽입, 삭제, 탐색의 시간복잡도는 O(log n)



    📌 사용 예

    • Java Collection의 ArrayList
    • HashMap의 Separate Chaining
    • C++ map



    📌 해시와 비교해서 본 장점

    • 순서가 있는 자료일 경우 좋다. ( 해쉬는 순서가 없음 )

    • 일관성있는 퍼포먼스를 보여준다. ( 해쉬는 rehashing시 비정상적 시간이 걸릴 수 있음 )

    • 해쉬는 페이지폴트를 일으킬 수 있다.

    • 연속된 삽입간의 공간 지역성을 유지하기 쉽다. ( 더 적은 I/O 발생)

    • 트리는 부정확한 검색에 사용될 수 있다.



    📌 구현


    Insert ( 삽입 )

     1BST 특징대로 삽입 후, 삽입 노드의 색깔을 RED로 설정.
 2삽입 후 RBT의 특징을 위배할 시 노드의 색깔을 조정하고,
 3Black-Height가 위배되었다면, rotation을 통하여 height을 조정.
 4
 5이때, 여러 case가 존재하는 데, 새로 삽입한 노드를 z라고 할때,
 6부모 노드인 p[z]가 부모 노드의 부모 노드인 할아버지 노드 p[p[z]]의 왼쪽 자식인지 오른쪽 자식인지에 따라 case가 나뉘게 된다.
 7두개의 경우가 서로 대칭이다.
 8
 9크게 보면 아래와 같이 2가지의 경우일 때, red-black tree의 규칙이 위배가 된다.
10
11🔴 case 1 : z 삼촌이 레드  ➡ 색상 변환 ( Recoloring )
12
13    - z의 부모와 z의 삼촌 노드를 레드에서 블랙으로 바꾸고, z의 할아버지 노드를 블랙에서 레드로 바꾼다.
14    - z의 할아버지 노드의 부모노드가 레드인 경우 이 경우를 반복 한다.
15    - z의 부모가 블랙을 만날때 종료 되며, 루트까지 올라가게 되면 루트노드를 블랙으로 바꾸고 종료한다. (루트노드까지 올라가게 되면 black-height는 1 증가하게 된다.)
16
17 🔴 case 2 : z의 삼촌이 없거나 블랙 ➡ 회전 ( rotation , restructuring)
18
19  ➡ 부모노드 (p[z])가 할아버지 노드 (p[p[z]]) 의 왼쪽 자식일때
20    - case 2-1 : z가 p[z]의 오른쪽 자식
21        -> p[z]를 중심으로 왼쪽으로 회전 시키고, 여전히 레드 블랙트리 특성을 위반하므로 case 2-2를 수행한다.
22    - case 2-2 : z가 p[z]의 왼쪽 자식
23        -> p[p[z]]를 중심으로 오른쪽 회전 시키고, p[z]와 p[p[z]]의 색상을 바꾼다. (부모노드는 블랙으로, 할아버지 노드는 레드로)
24
25  ➡ 부모노드 (p[z])가 할아버지 노드 (p[p[z]]) 의 오른쪽 자식일때
26    - case 2-1 : z가 p[z]의 왼쪽 자식
27        -> p[z]를 중심으로 오른쪽으로 회전 시키고, 여전히 레드 블랙트리 특성을 위반하므로 case 2-2를 수행한다.
28    - case 2-2 : z가 p[z]의 오른쪽 자식
29        -> p[p[z]]를 중심으로 왼쪽 회전 시키고, p[z]와 p[p[z]]의 색상을 바꾼다. (부모노드는 블랙으로, 할아버지 노드는 레드로)
30
31z를 할아버지 노드 ( p[p[z]] )로 바꿔주고 할아버지의 부모가 Red가 아닐때까지 위의 case를 반복해준다.



    Delete (삭제)

     1 BST의 특성을 유지하면서 삭제한 후, 삭제할 노드의 자식노드 개수에 따라 rotation 방법이 달라진다.
 2 또한, 지워진 노드의 색깔이 Black이라면, Black-Height가 1감소한 경로에 black node가 1개 추가되도록 rotaion한다.
 3
 4 NULL node(leaf node) 역시 black 이다.
 5
 6
 7 🔴 case default : 삭제할 노드를 z라 할때, z가 RED라면 그냥 삭제하고, (z의 자식이 두개인 경우는 오른쪽 자식의 가장 작은 key와 key값을 교환 후 z->right의 minimum 노드를 삭제하게 되므로 이 노드의 색이 RED라면)
 8 BLACK이라면,  black-height가 안맞게 되므로 fix up을 수행한다. (이때, double-black개념이 등장한다.)
 9
10 삭제도 삽입과 마찬가지로 삭제한 노드 z대신 위치할 노드 x가 p[x]의 왼쪽 자식인지 오른쪽 자식인지에 대해 대칭한다.
11
12 삭제한 노드 z대신 새로 위치한 노드를 x, 그 형제 노드를 s라고 할때,
13 🔴 case 1 : s가 RED인 경우 ➡
14     이때는 s의 자식들은 leafNode 일 수 없다.(조건 5 위반) => 한개라도 leafnode일 시 black-heigh가 달라지므로 무조건 두개를 가지고 있다.
15
16   -> x가 부모노드 p[x]의 왼쪽 자식일때
17     - s를 BLACK으로 p[x]를 RED로 바꿔준다.
18     - p[x]를 left-Rotate 시켜준다.
19     - x의 새로운 형제노드를 달아준다. (s = p[x]->right) ==> x의 새로운 형제노드는 원래 s의 왼쪽 자식노드 (left-Rotate시켜주었기 때문)
20
21   -> x가 부모노드 p[x]의 오른쪽 자식일때
22     - s를 BLACK으로 p[x]를 RED로 바꿔준다.
23     - p[x]를 right-Rotate 시켜준다.
24     - x의 새로운 형제노드를 달아준다. (s = p[x]->left) ==> x의 새로운 형제노드는 원래 s의 왼쪽 자식노드 (left-Rotate시켜주었기 때문)
25
26   아직 double-black이 남았기 때문에 case 2/3/4를 진행한다.
27
28 🔴 case 2 :s가 BLACK, s의 자식들도 BLACK일때 ➡
29     - x의 double-blackd을 지우고 s를 RED로 바꾼다.
30     - p[x]를 x로 해서 계속 한다.
31     - 만약, case 1을 거치고 case 2로 왔다면, p[x]는 red였기때문에 종료된다. ( => s의 자식들이 모두 black인 채로 있기 때문에 black-height는 유지)
32
33 🔴 case 3 : s는 BLACK, s의 왼쪽 자식이 RED, 오른쪽 자식이 BLACK 인 경우 ➡
34
35    -> x가 부모노드 p[x]의 왼쪽 자식일때
36     - s를 RED, s의 왼쪽 자식을 BLACK으로 바꿔준다.
37     - s를 중심으로 right-Rotate 시켜준다.
38     - x의 새로운 형제노드를 달아준다. ( s = p[x]->right)
39
40    -> x가 부모노드 p[x]의 오른쪽 자식일때
41     - s를 RED, s의 왼쪽 자식을 BLACK으로 바꿔준다.
42     - s를 중심으로 left-Rotate 시켜준다.
43     - x의 새로운 형제노드를 달아준다. ( s = p[x]->left)
44
45     case 3의 경우는 위의 과정을 끝내고 case 4를 이어서 수행한다.
46
47 🔴 case 4 : s는 BLACK, s의 오른쪽 자식이 RED인 경우 ➡
48
49   -> x가 부모노드 p[x]의 왼쪽 자식일때
50     - s의 색을 p[x]의 색으로 바꿔준다.
51     - p[x]의 색을 BLACK, s의 오른쪽 자식을 BLACK으로 바꿔준다.
52     - p[x]에 대해서 left-rotate를 시켜준다.
53     - x의 double-black을 제거하고 종료한다.
54
55   -> x가 부모노드 p[x]의 오른쪽 자식일때
56     - s의 색을 p[x]의 색으로 바꿔준다.
57     - p[x]의 색을 BLACK, s의 왼쪽 자식을 BLACK으로 바꿔준다.
58     - p[x]에 대해서 right-rotate를 시켜준다.
59     - x의 double-black을 제거하고 종료한다.
60
61 x가 root이거나 double black이 깨질때까지 반복해준 후, x를 black으로 바꿔준다.



    Search (탐색)

    1Red Black Tree는 BST의 일종이기 때문에 탐색의 밥법은 일반적인 Bianry Tree의 탐색 방법과 다르지 않다.
2
3찾는 값이 해당 노드보다 작다면 왼쪽으로 크다면 오른쪽으로 내려가며 값을 탐색

    📃 구현 코드 (c++) ( 접기 / 펼치기 ) 
      1/*
  2* C++ 이용하여 Red Black Tree 구현하기
  3*
  4* 목적 : Red Black Tree 공부 하기 위해 작성했으며,
  5*       C++ 이용하여 작성하시는 분들에게 도움이 되고자 했다.
  6*

  8*       이중 연결 리스트로 구현
  9*       Red / Black은 식별하기 쉽게 enum이용 했으며, bool 이용시 데이터 크기 절약
 10*
 11*       class RBTree
 12*
 13*       변수 :   root node => root노드는 항상 black
 14*               leaf node => 끝에 해당하는 노드들은 leaf node들을 가지고 있다.
 15*                            leaf node라는 것만 알면 되기 때문에 새로운 노드 삽입 때마다 leaf node를 생성 해줄 필요없이
 16*                            모든 말단 노드들은 이 leaf node를 가리키는 식으로 구현
 17*                            leaf node는 항상 black
 18*
 19*       생성자 : RBTREE =>  node 구조체 생성후
 20*                          색은 black 초기화
 21*                          모든 자식은 nullptr로 초기화.
 22*
 23*       함수 :   IsKey => key값이 있는지 검사하는 함수
 24*               Insert => 삽입 함수
 25*               InsertFixUp => 삽입 후 규칙 깨졌을 시 재조정 함수
 26*               Delete => 삭제 함수
 27*               DeleteFixUp => 삭제 후 규칙 깨졌을 시 재조정 함수
 28*               Transplant => 삭제 시 이용하며, 삭제할 노드의 자식 노드를 부모노드에 연결해주는 함수
 29*               RotateRight(x) => x기준 오른쪽으로 회전
 30*               RotateLeft(x) => x기준 왼쪽으로 회전
 31*
 32*               Inorder,Preorder,Postorder => 순회 함수
 33*               tree_minimum(x), tree_maximum(x) => 노드 x 기준으로 가장 왼쪽, 오른쪽 return 함수
 34*
 35*               DisplayMenu, SelectMenu => 초기 Menu판 print 함수
 36*               Insert_helper,Delete_helper,order_helper,print_helper => 각각 수행시 입력받고 조건 에러 처리 위한 함수 와 tree print 해주는 함수

 38* 작성자 : gowoonsori

 40* 해당 source gist : https://gist.github.com/gowoonsori/a725e29ef1880f0592fe5760f4908c6b
 41*/

 43#include <iostream>
 44
 45enum Color {
 46    RED,
 47    BLACK
 48};
 49struct node {
 50    int key;
 51    node *left = nullptr;
 52    node *right = nullptr;
 53    node *parent = nullptr;
 54    Color color = BLACK;
 55};
 56
 57typedef node *NodePtr;
 58
 59class RBTREE {
 60   private:
 61    NodePtr root;      //루트 노드
 62    NodePtr leafNode;  //단말노드
 63
 64    //key값이 있는지 없는지 검사 있으면 pointer 값, 없으면 nullptr
 65    NodePtr IsKey(int item) {
 66        NodePtr t = root;
 67        NodePtr parent = NULL;
 68
 69        /*key값을 찾거나 없다면 break*/
 70        while (t != NULL && t->key != item) {
 71            parent = t;
 72            t = (item < parent->key) ? parent->left : parent->right;
 73        }
 74        return t;
 75    }
 76
 77    void Insert(int item) {
 78        // x : 삽입할 곳 찾기위한 포인터 | y : 삽입할 곳의 부모노드
 79        NodePtr x = this->root, y = nullptr;
 80        NodePtr z = new node();
 81        z->key = item;
 82        z->color = RED;
 83        z->parent = nullptr;
 84        z->left = leafNode;
 85        z->right = leafNode;
 86
 87        /*BST의 일반 삽입 연산*/
 88        while (x != leafNode) {
 89            y = x;
 90            if (x->key < item)
 91                x = x->right;
 92            else
 93                x = x->left;
 94        }
 95
 96        z->parent = y;
 97
 98        if (y == nullptr)
 99            root = z;
100        else if (z->key > y->key)
101            y->right = z;
102        else
103            y->left = z;
104
105        //z가 root노드라면
106        if (z->parent == nullptr) {
107            z->color = BLACK;
108            return;
109        }
110        // z의 부모노드가 root노드라면 Fix Up 필요없이 red컬러로 붙여주면 된다.
111        if (z->parent->parent == nullptr) {
112            return;
113        }
114        InsertFixUp(z);
115
116        return;
117    }
118
119    void InsertFixUp(NodePtr z) {
120        /*root 노드가 아니고 부모 색이 red라면*/
121        while (z != root && z->parent->color == RED) {
122            NodePtr grandparent = z->parent->parent;
123            NodePtr uncle = (z->parent == grandparent->left) ? grandparent->right : grandparent->left;
124            bool side = (z->parent == grandparent->left) ? true : false;  //if p[z]가 p[p[z]]의 왼쪽 자식이면 1 / 오른쪽이면 0
125
126            /*case 1*/
127            if (uncle && uncle->color == RED) {
128                z->parent->color = BLACK;
129                uncle->color = BLACK;
130                grandparent->color = RED;
131                z = grandparent;
132            }
133
134            /*case 2
135                side == true ) p[z]는 p[p[z]]의 왼쪽 자식 인 경우이다.
136                side == false ) p[z]는 p[p[z]]의 오른쪽 자식 인 경우이다. */
137            else {
138                /*case 2-1*/
139                if (z == (side ? z->parent->right : z->parent->left)) {
140                    z = z->parent;
141                    side ? RotateLeft(z) : RotateRight(z);
142                }
143                /*case 2-2*/
144                z->parent->color = BLACK;
145                grandparent->color = RED;
146                side ? RotateRight(grandparent) : RotateLeft(grandparent);
147            }
148        }
149        root->color = BLACK;
150    }
151
152    bool Delete(int item) {
153        NodePtr z = IsKey(item);
154        if (!z)
155            return false;
156        else {
157            NodePtr x, y;
158            Color OriginalColor = z->color;
159
160            /*자식이 없거나 1개인 경우
161                    삭제할 노드(z)가 블랙이면 doulbe red이므로 fix*/
162            if (z->left == leafNode) {
163                x = z->right;
164                Transplant(z, z->right);
165            } else if (z->right == leafNode) {
166                x = z->left;
167                Transplant(z, z->left);
168            } else {
169                y = tree_minimum(z->right);
170                OriginalColor = y->color;
171                x = y->right;  //y의 왼쪽 자식은 없다.
172
173                if (y->parent == z) {  //z의 오른쪽 자식이 가장 작은 key
174                    x->parent = y;     // x가 leafnode일 때, fix하게 될 때 사용
175                } else {
176                    Transplant(y, y->right);
177                    y->right = z->right;
178                    y->right->parent = y;
179                }
180                Transplant(z, y);
181                y->left = z->left;
182                y->left->parent = y;
183                y->color = z->color;
184            }
185            delete z;
186            if (OriginalColor == BLACK) {
187                DelteFixUp(x);
188            }
189        }
190        return true;
191    }
192
193    void DelteFixUp(NodePtr x) {
194        NodePtr s;  //형제노드 s
195
196        //root이거나 double black 이 깨질때 까지
197        while (x != root && x->color == BLACK) {
198            /* x가 p[x]의 왼쪽자식인 경우 */
199            if (x == x->parent->left) {
200                s = x->parent->right;
201                // case 1
202                if (s->color == RED) {
203                    s->color = BLACK;
204                    x->parent->color = RED;
205                    RotateLeft(x->parent);
206                    s = x->parent->right;
207                }
208
209                // case 2
210                if (s->left->color == BLACK && s->right->color == BLACK) {
211                    s->color = RED;
212                    x = x->parent;
213                } else {
214                    // case 3
215                    if (s->right->color == BLACK) {
216                        s->left->color = BLACK;
217                        s->color = RED;
218                        RotateRight(s);
219                        s = x->parent->right;
220                    }
221
222                    // case 4
223                    s->color = x->parent->color;
224                    x->parent->color = BLACK;
225                    s->right->color = BLACK;
226                    RotateLeft(x->parent);
227                    x = root;
228                }
229            }
230
231            /*x가 p[x]의 오른쪽 자식인 경우*/
232            else {
233                s = x->parent->left;
234                // case 1
235                if (s->color == RED) {
236                    s->color = BLACK;
237                    x->parent->color = RED;
238                    RotateRight(x->parent);
239                    s = x->parent->left;
240                }
241
242                // case 2
243                if (s->left->color == BLACK && s->right->color == BLACK) {
244                    s->color = RED;
245                    x = x->parent;
246                } else {
247                    // case 3
248                    if (s->left->color == BLACK) {
249                        s->right->color = BLACK;
250                        s->color = RED;
251                        RotateLeft(s);
252                        s = x->parent->left;
253                    }
254                    // case 4
255                    s->color = x->parent->color;
256                    x->parent->color = BLACK;
257                    s->left->color = BLACK;
258                    RotateRight(x->parent);
259                    x = root;
260                }
261            }
262        }
263        x->color = BLACK;
264        root->color = BLACK;
265    }

267    /* u의 위치에 v를 이식 */
268    void Transplant(NodePtr u, NodePtr v) {
269        if (u->parent == nullptr)
270            root = v;
271        else if (u == u->parent->left)
272            u->parent->left = v;
273        else
274            u->parent->right = v;
275
276        v->parent = u->parent;
277    }
278    /*x를 중심으로 왼쪽으로 회전*/
279    void RotateLeft(NodePtr x) {
280        NodePtr y;
281
282        y = x->right;
283        x->right = y->left;
284        if (y->left != leafNode) {
285            y->left->parent = x;
286        }
287        y->parent = x->parent;
288
289        if (!x->parent) {
290            root = y;
291        } else if (x == x->parent->left) {
292            x->parent->left = y;
293        } else {
294            x->parent->right = y;
295        }
296        x->parent = y;
297        y->left = x;
298    }
299    /*x를 중심으로 오른쪽으로 회전*/
300    void RotateRight(NodePtr y) {
301        NodePtr x;
302
303        x = y->left;
304        y->left = x->right;
305        if (x->right != leafNode) {
306            x->right->parent = y;
307        }
308        x->parent = y->parent;
309
310        if (!y->parent) {
311            root = x;
312        } else if (y == y->parent->left) {
313            y->parent->left = x;
314        } else {
315            y->parent->right = x;
316        }
317        y->parent = x;
318        x->right = y;
319    }

336            print_helper(root->left, indent, false);

338        }
